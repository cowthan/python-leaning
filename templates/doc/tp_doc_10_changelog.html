更新日志

2017年11月9日

  • TinkerPatch 1.2.1 发布
  • 基于 Tinker 1.9.1
  • 功能: 支持实验功能supportComponent,支持动态新增Activity
  • 功能： 支持android gradle plugin 3.0
  • Bugfix: 修复Android O patch过程的一个bug
  • Bugfix: TinkerMultidexConfigTask: FileNotFoundException(manifest_keep.txt)#649
  • Bugfix: 资源合成时需要保持原来的压缩方式
  • Bugfix: 修改判断是否为主进程的方式

2017年8月10日

  • TinkerPatch 1.1.8 发布
  • 基于 Tinker 1.8.1
  • 功能: 支持Android O
  • 功能: 在tinkerpatchSupport中增加backupFileNameFormat，来控制备份目录的文件名
  • 优化: 补丁后 Android N启动速度
  • Bugfix: 修复RxJava 可能出现的verify error
  • Bugfix: 修复部分CA签名校验不过的问题
  • Bugfix: 修复patchversion没有赋值的问题
  • 重构: 在ART平台合并多个classesN.dex到同一个文件，减少多线程dex2oat对系统的影响

2017年5月24日

  • TinkerPatch 1.1.7 发布
  • 基于 Tinker 1.7.11
  • 新增指定requestLoader的接口，可以传入自己实现或者第三方的Httpclient作为tinkerPatch请求的client，文档 11. 定制加载器模块
  • 新增Okhttp3Loader和OkhttpLoader
  • Bugfix: 修复1.7.10 版本替换AndroidNClassloader时可能产生found duplicate classes warning
  • Bugfix: 修复自动keep main dex pattern中的问题，将() 改为 (...)
  • Bugfix: 修复坚果Pro 资源Patch异常
  • Bugfix: 修复360加固 Android N上的问题
  • Bugfix: 修复Android O so 反射问题

2017年5月07日

  • TinkerPatch 1.1.6 发布
  • 基于 Tinker 1.7.9-5f996c9
  • Bugfix: Class XXX has already interned.
  • Bugfix: auto force-jumbo mode has no effect when keepDexApply option was enabled. 避免不必要的loader class changed告警
  • Bugfix: loader class pattern has no effect in 'protected app' mode.
  • Bugfix: NPE when isProtectedApp option is enabled.

2017年4月26日

  • TinkerPatch 1.1.5 发布
  • 基于 Tinker 1.7.9
  • 支持加固App的patch生成，具体请参照
  • 修复了集成了趣拍sdk的app无法生成patch的问题
  • 增加了合成成功的上报
  • 增加fetchPatchUpdateAndPollWithInterval接口用于轮询更新

2017年2月26日

  • TinkerPatch 1.1.4 发布
  • 基于 Tinker 1.7.7
  • 增加上报重试保证准确性
  • 兼容使用 Java 7 编译
  • 重构了若干 Log

2017年2月7日

  • TinkerPatch 1.1.3 发布
  • 基于 Tinker 1.7.7
  • 增加cleanAll接口，用户清除所有tinkerPatch存储的信息
  • 重构了若干 Log

2017年1月16日

  • TinkerPatch 1.1.1 发布
  • 基于 Tinker 1.7.7
  • 重构 ProductFlavors 字段
  • 兼容 1.0.1 TinkerPatchSupport 字段
